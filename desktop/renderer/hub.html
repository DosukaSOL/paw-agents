Create a Telegram agent with BotFather

Spin up a fully working PAW agent on Telegram in under five minutes. PAW is the backend — no webhook server, no Flask, no cloud hosting required.

  1. 1Open BotFather on Telegram
    Search for @BotFather in Telegram or tap here to open it. Press Start to begin a chat.
  2. 2Create a new bot
    Send /newbot. BotFather will ask for two things:
    • Display name — what people see in chats (e.g. My PAW Agent).
    • Username — must end in bot (e.g. mypawagent_bot).
  3. 3Copy your bot token
    BotFather replies with a token that looks like: 
    123456789:AAH-thisIsYourSecretBotTokenKeepItSafe
    Treat this like a password — anyone with it can control your bot.
  4. 4Add the token to PAW
    Open .env at the root of your PAW install and add: 
    TELEGRAM_BOT_TOKEN=123456789:AAH-thisIsYourSecretBotTokenKeepItSafe
    Or set it from Settings → Channels.
  5. 5Restart PAW
    Stop and re-run node dist/index.js (or pm2 restart paw). On boot you should see: 
    [PAW] 🐾 Starting Telegram bot...
[PAW] ✓ Telegram bot @mypawagent_bot online
  6. 6Say hi to your bot
    Open Telegram, find your bot by username, send /start — your PAW agent will reply using whichever model provider you configured.
  7. 7(Optional) Polish it with BotFather
    Back in @BotFather you can:
    • /setdescription — short pitch shown in the chat preview.
    • /setabouttext — “About” section on your bot's profile.
    • /setuserpic — upload a profile picture.
    • /setcommands — register slash commands like /help, /reset.
    • /setjoingroups — allow your bot to be added to groups.
    • /setprivacy — set to Disable if you want it to read all group messages.
Tip: for 24/7 uptime use pm2, systemd, launchd, or Docker — but the “backend” itself is already PAW. You don't need a separate webhook server.
